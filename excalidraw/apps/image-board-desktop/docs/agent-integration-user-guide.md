# CoreStudio Agent 集成使用说明

Agent 集成是一组本地协作能力。它不是把 CoreStudio 变成一个内置 Agent 平台，而是让外部 Agent 能通过本地客户端安全地查看、读取和写回项目。

核心原则：

- CoreStudio 桌面端仍然是项目数据 owner。
- 本地客户端必须运行，Agent Board、CLI 和 ACP Agent 才能工作。
- 写回项目必须经过 CoreStudio CLI / Local Bridge，不直接改项目文件。
- 图片资产、生成记录、画板元素和 Agent 对话记录应能互相解释。

## 三条使用路径

| 路径 | 什么时候用 | 谁发起 | 结果在哪里看 |
| --- | --- | --- | --- |
| 网页画布 | 在 Codex、Cursor 或其他 Agent 内置浏览器里查看和操作当前项目画板 | 用户复制或打开 Agent Board 链接 | 画布、生成记录、右下角状态浮层 |
| CLI | 让 Agent 自动读取选区、图片原始路径、项目健康状态，或写入新图片 | Agent 或用户 shell | 画布、生成记录、项目健康报告 |
| ACP Agent | 从 CoreStudio 主动发起复杂任务给外部 Agent，并继续对话 | CoreStudio 底部输入框 | 左侧 Agent 对话、画布、生成记录 |

## 入口怎么分工

| 入口 | 负责什么 | 不负责什么 |
| --- | --- | --- |
| 应用设置 | 开启 Agent 集成，配置 ACP Agent，查看网页画布和 CLI 用法，进入高级调试 | 日常任务历史 |
| 右下角状态浮层 | 查看 Bridge、当前项目、网页画布、CLI、ACP Agent 是否可用，并复制快捷信息 | 配置表单和任务列表 |
| 底部输入框 | 快速发起直接输入或 ACP Agent 任务 | 完整历史和原始调试 JSON |
| 左侧栏 | 查看生成记录、ACP thread、工具调用、图片结果，并继续对话 | 应用设置 |
| 画布主菜单 | 查看当前项目、切换项目和使用 Excalidraw 原生画布动作 | 项目维护、ACP 调试记录 |
| 桌面应用菜单 | 新建、打开、最近项目、项目维护、项目数据检查和修复、应用设置 | 画布内过程记录 |
| 高级调试 | 查看 ACP run log、协议 JSON、任务包和错误详情 | 普通用户主流程 |

## 直接输入和 ACP Agent 的区别

### 直接输入

直接输入是单次生成模式。

- 适合一句 prompt 生成一张或几张图。
- 每次生成之间默认没有连续上下文。
- 左侧栏显示的是生成记录列表。
- 结果应该能定位到画布里的图片元素。

### ACP Agent

ACP Agent 是连续任务模式。

- 适合复杂任务，例如读取参考图、分析选区、生成方案、调用 CLI 写回。
- 同一个 thread 里可以继续对话。
- 左侧栏显示 Agent 回复、工具调用和图片结果。
- 图片结果应带来源、prompt 摘要、参考关系，并能定位到画布。

## 网页画布怎么用

1. 在 CoreStudio 中开启 Agent 集成。
2. 打开本地项目。
3. 从右下角状态浮层或设置页复制 Agent Board 链接。
4. 在 Codex、Cursor 或其他 Agent 内置浏览器中打开链接。
5. 在画布里选择图片或标注内容，让 Agent 通过 CLI / Bridge 读取和写回。

注意：

- 网页画布依赖本地 CoreStudio 运行，不能脱离客户端独立工作。
- 网页画布可以独立切换项目，但仍然通过本地 Bridge 读取项目数据。
- 如果画布显示连接异常，先看右下角状态浮层里的 Bridge 和当前项目状态。

## CLI 怎么用

CLI 是给 Agent 和高级用户使用的稳定命令入口。命令心智固定为四类：

- `read`：读取状态、项目、选区、图片路径、生成记录、健康报告。
- `write`：写入图片、prompt 或生成记录。
- `edit`：执行临时定位、选择等画布操作。
- `bash`：输出环境变量和可复制示例。

通过右下角状态浮层复制 CLI 环境变量后，Agent 可以在 shell 里自动找到当前 Bridge 和项目 token。

写入要求：

- 来源不能为空。
- 图片文件必须可读取。
- 生成图片要能写入 image record、generation record 和画板元素。
- 写入失败应返回结构化错误，而不是留下半截数据。

## ACP Agent 怎么用

1. 在应用设置里启用 ACP Agent。
2. 选择或填写外部 Agent 命令、参数和工作目录。
3. 在底部输入框切到 ACP Agent。
4. 输入任务，点击发送。
5. 在左侧 Agent 对话里查看 thread、工具调用和图片结果。
6. 需要继续调整时，在左侧栏底部继续对话。

ACP Agent 的职责边界：

- CoreStudio 负责发起任务、传递项目上下文、保存 thread 和展示结果。
- 外部 Agent 负责分析、生成、调用 CLI。
- 写回仍然必须通过 CoreStudio CLI / Local Bridge。
- CoreStudio 不内置 Agent 调度器。

## 项目健康检查和修复

如果出现图片资产存在但画布找不到、生成记录定位失败、ACP 结果没有写回等问题，使用项目维护里的项目数据检查和修复。

健康报告应说明：

- 发现了什么问题。
- 影响是什么。
- 能不能自动修复。
- 修复时会做什么。
- 如果跳过，为什么跳过。

修复功能不是单纯修缩略图。它会尽量让项目资产、画板元素、生成记录和 ACP thread 重新对齐。

## 常见问题

### 右下角显示 Bridge 未连接

确认 CoreStudio 开发版或正式版正在运行，并且应用设置里开启了 Agent 集成。

### 网页画布打不开项目

先确认本地客户端已启动，再确认链接里的 Bridge 地址可访问。项目 token 是项目固定信息，端口可能因为占用而变化。

### ACP Agent 任务完成了，但没有图片

先看左侧 Agent 对话里的工具调用和错误信息。如果需要原始协议细节，再到应用设置的高级调试查看 run log。

### 生成记录能看到，但点击无法定位

运行项目数据检查。如果图片资产存在但画板没有元素，项目数据修复会尝试补回画板元素。

### 直接输入和 ACP Agent 应该怎么选

简单单次生成用直接输入。需要读参考图、连续修改、调用工具或让外部 Agent 自己判断步骤时，用 ACP Agent。
