# CoreStudio Codex 集成使用说明

“Codex 集成”让 Codex 通过本地客户端安全地查看、读取和写回 CoreStudio 项目。它不是一个需要保持“连接”的账户功能，也不需要额外的启停开关。

核心原则：

- CoreStudio 桌面端仍然是项目数据 owner。
- 任务在哪里发起，哪里就是本次任务的调度者。
- 本地客户端必须运行，Codex 内置画布、CLI 和实验性 ACP 才能访问项目。
- 写回项目必须经过 CoreStudio CLI / Local Bridge，不直接改项目文件。
- 在 Codex 中发起生成时，默认使用 Codex 自身的生图能力。

## 默认使用路径

| 路径 | 什么时候用 | 调度者 | 数据如何落回 CoreStudio |
| --- | --- | --- | --- |
| CoreStudio 直接生成 | 在 CoreStudio 内使用已配置的模型 API 生图 | CoreStudio | CoreStudio 内部直接保存图片、记录和画布元素 |
| 在 Codex 中使用 CoreStudio | 在 Codex 对话中查看画布、读取选区、生成并写回结果 | Codex | CLI / Local Bridge |

CLI / Local Bridge 是 Codex 协作的后台数据通道，不是普通用户需要单独学习的一条产品路径。网页画布是视觉上下文，也不再提供另一套生成输入器。

## 入口怎么分工

| 入口 | 负责什么 | 不负责什么 |
| --- | --- | --- |
| 应用设置 · 图像生成 | 选择当前服务和模型，配置各服务 API Key | Agent 连接状态 |
| 应用设置 · Codex 集成 | 提供自然语言安装请求并检测 CLI、CoreStudio Skill 和会话发现能力 | 允许连接开关、已连接状态和内部脚本参数 |
| 应用设置 · 实验性功能 | 手动开启 ACP，并按需进入高级配置 | 默认工作流 |
| CoreStudio 底部输入框 | 发起 CoreStudio 直接生成；开启实验性 ACP 后可切换外部 Agent | Codex 中的任务 |
| 左侧栏 | 默认查看生成记录；开启实验性 ACP 后查看 thread、工具调用和图片结果 | 应用设置和原始调试 JSON |
| 画布主菜单 | 查看当前项目、切换项目和使用 Excalidraw 原生画布动作 | 项目维护、ACP 调试记录 |
| 桌面应用菜单 | 新建、打开、最近项目、项目维护、项目数据检查和修复、应用设置 | 画布内过程记录 |
| 高级调试 | 查看 ACP run log、协议 JSON、任务包和错误详情 | 普通用户主流程 |

应用设置是真正的配置中心。菜单只保留“应用设置”入口；画布右下角不再展示 Codex 状态浮层。Codex 集成页只说明依赖是否安装完整，并提供一句可以直接发给 Codex 的自然语言安装请求。安装脚本和路径参数只写在应用自带的 Agent 指南中，不直接暴露给用户。

## CoreStudio 直接生成

直接输入是单次生成模式。

- 适合一句 prompt 生成一张或几张图。
- 每次生成之间默认没有连续上下文。
- 左侧栏显示的是生成记录列表。
- 结果应该能定位到画布里的图片元素。

## 在 Codex 中使用 CoreStudio

1. 打开 CoreStudio 的“应用设置 → Codex 集成”。
2. 复制页面给出的自然语言安装请求并发给 Codex，让 Codex 阅读应用自带指南并完成安装，然后点击“重新检测”。
3. 检测通过后，复制“打开当前 CoreStudio 项目”的提示词，在 Codex 中发送。
4. Codex 打开当前画布后，可以查看、选择或标注参考内容，再在对话中发起任务。
5. Codex 默认使用自身的生图能力，并通过 CLI / Local Bridge 写回图片、生成记录和画布元素。

注意：

- 网页画布依赖本地 CoreStudio 运行，不能脱离客户端独立工作。
- 网页画布仍然通过本地 Bridge 读取项目数据，但画布内不提供生成输入器或 ACP 调度入口。
- 如果 Codex 无法打开项目，回到“Codex 集成”页面重新检测依赖，再把页面给出的自然语言请求发给 Codex 处理。

## CLI 怎么用

CLI 是给 Agent 和高级用户使用的稳定命令入口。命令心智固定为四类：

- `read`：读取状态、项目、选区、图片路径、生成记录、健康报告。
- `write`：写入图片、prompt 或生成记录。
- `edit`：执行临时定位、选择等画布操作。
- `bash`：输出环境变量和可复制示例。

CLI 通常由 CoreStudio Skill 帮助 Codex 自动发现当前会话。普通用户只需要把“Codex 集成”页面提供的一句话发给 Codex，不需要接触安装脚本或手动管理环境变量。

写入要求：

- 来源不能为空。
- 图片文件必须可读取。
- 生成图片要能写入 image record、generation record 和画板元素。
- 写入失败应返回结构化错误，而不是留下半截数据。

## 实验性功能：外部 Agent（ACP）

ACP 只用于“从 CoreStudio 内部把任务交给外部 Agent”，默认关闭。它不是在 Codex 中使用 CoreStudio 的必经路径。

1. 在应用设置的“实验性功能”中开启“外部 Agent（ACP）”。
2. 选择或填写外部 Agent 命令、参数和工作目录。
3. 在底部输入框切到 ACP Agent。
4. 输入任务，点击发送。
5. 在左侧 Agent 对话里查看 thread、工具调用和图片结果。
6. 需要继续调整时，在左侧栏底部继续对话。

ACP Agent 的职责边界：

- CoreStudio 负责发起任务、传递项目上下文、保存 thread 和展示结果。
- 外部 Agent 负责分析、生成、调用 CLI。
- 写回仍然必须通过 CoreStudio CLI / Local Bridge。
- CoreStudio 不内置 Agent 调度器。
- 关闭实验性功能不会删除已有配置、thread 或运行记录。
- Codex 内置画布不使用 ACP，避免形成 `Codex → CoreStudio → ACP → Codex` 回路。

## 项目健康检查和修复

如果出现图片资产存在但画布找不到、生成记录定位失败、ACP 结果没有写回等问题，使用项目维护里的项目数据检查和修复。

健康报告应说明：

- 发现了什么问题。
- 影响是什么。
- 能不能自动修复。
- 修复时会做什么。
- 如果跳过，为什么跳过。

修复功能不是单纯修缩略图。它会尽量让项目资产、画板元素、生成记录和 ACP thread 重新对齐。

## 常见问题

### Codex 集成检测未通过

复制页面当前给出的自然语言请求并发给 Codex。Codex 会阅读应用自带指南，完成安装、更新或修复。回到 CoreStudio 点击“重新检测”，三项依赖都通过后，再把“打开当前 CoreStudio 项目”的提示词发给 Codex。

### 网页画布打不开项目

先确认本地客户端已启动，再确认链接里的 Bridge 地址可访问。项目 token 是项目固定信息，端口可能因为占用而变化。

### ACP Agent 任务完成了，但没有图片

先看左侧 Agent 对话里的工具调用和错误信息。如果需要原始协议细节，再到应用设置的高级调试查看 run log。

### 生成记录能看到，但点击无法定位

运行项目数据检查。如果图片资产存在但画板没有元素，项目数据修复会尝试补回画板元素。

### CoreStudio 直接生成和实验性 ACP 应该怎么选

普通生图使用 CoreStudio 直接生成。在 CoreStudio 内确实需要把复杂任务交给外部 Agent 时，再手动开启实验性 ACP。在 Codex 中工作时直接从 Codex 对话发起，不需要开启 ACP。
