## Design Context

### Users
CoreStudio is a local Electron image board built on Excalidraw for industrial design exploration. Users are designers and product builders who want to generate, edit, compare, and trace AI image directions directly on a canvas without leaving the visual workspace.

### Brand Personality
The interface should feel precise, native, and quietly capable. It should borrow the lightness and hand-drawn friendliness of Excalidraw while avoiding rough prototype UI, generic admin forms, and visually heavy modal patterns.

### Aesthetic Direction
Use a refined native-tool direction: compact surfaces, clear alignment, soft neutral backgrounds, deliberate typography, and subtle interaction feedback. Avoid decorative gradients, heavy shadows, glass effects, oversized radii, and dense nested cards. Advanced configuration should feel like professional progressive disclosure, not developer settings leaking into the main flow.

### Design Principles
- Preserve the canvas as the main stage; configuration should support creation without stealing attention.
- Keep the primary path short and human-readable; hide technical details until the user asks for them.
- Match Excalidraw's native rhythm: modest radius, clean outlines, soft contrast, and small purposeful controls.
- Use visual grouping instead of form clutter; related fields should read as one decision.
- Every label should explain user intent, not internal implementation.

### Iteration Principles
- Prefer reuse: when Excalidraw already has a similar component, interaction pattern, or visual treatment, extend or reuse that native language first.
- Redefine only when needed: when CoreStudio adds behavior the base product does not have, define the new component locally, but keep its spacing, radius, color weight, elevation, and motion within Excalidraw's visual range.
- Avoid visual forks: custom features should feel like they belong to the same canvas tool, not like a separate design system layered on top.
