# CoreStudio 应用设置重构设计

## 文档状态

- 状态：已确认，待编写实施计划
- 确认日期：2026-07-14
- 范围：CoreStudio 桌面端的统一应用设置、Codex 集成入口和画布上的 Codex 状态入口

## 背景

当前“应用设置”实际上只承载 Agent 集成相关内容，并将 Codex 协作、Bridge、网页画布、CLI 和 ACP 等底层概念平铺给用户。ACP 开启后还会立即展示命令、参数、工作目录和任务模板，使普通设置页看起来像开发者控制台。

本次重构不再继续压缩原有长页，而是将它重建为 CoreStudio 真正的统一设置中心。

## 与既有文档的关系

本文档继承“任务在哪里发起，哪里就是调度者”以及“ACP 属于实验性功能”等既有边界。对于设置页、Codex 集成入口和画布右下角状态浮层，本文档替代早期文档中的以下设计：

- 不再由应用设置提供“Codex 协作”权限开关。
- 不再将“已可用”、“已连接”和当前项目作为画布常驻状态。
- Codex 集成采用“复制指令、用户执行、CoreStudio 检测”的轻量流程，不由 CoreStudio 直接执行安装、更新或修复。

## 设计目标

1. 将分散的图像生成配置、Codex 集成和实验性功能收入同一设置入口。
2. 让每个分类只回答一类问题，不再将技术组件包装成平级的用户路径。
3. 将 Codex 集成改造为“安装、检测、使用”引导，而不是权限开关或连接状态面板。
4. 保留多个图像生成服务的能力，但默认只突出当前服务和配置结论。
5. 使 ACP 继续作为需要手动开启的实验性功能，默认不暴露技术配置。
6. 移除画布上不再有必要的 Codex 常驻状态入口，让画布回到内容优先。

## 非目标

- 不在设置中实现 Codex 深链接、自动拉起或预填任务。
- 不将 ACP 放入“Codex 集成”，也不将 ACP 安装与 Codex 默认集成合并。
- 不把 Board 链接、CLI 环境变量、Bridge token 等内部概念放在默认页面。
- 不在设置窗口底部提供全局“保存全部”。
- 不在本设计中扩展新的 Agent 调度或图像生成能力。

## 总体信息架构

应用设置继续在当前画布内以模态窗口打开，关闭后直接回到原画布。窗口采用较大尺寸，左侧是始终可见的分类导航，右侧只展示当前分类。

第一版包含三个一级分类：

1. **图像生成**
2. **Codex 集成**
3. **实验性功能**

首次打开默认进入“图像生成”。在同一次应用使用期间，设置窗口记住上次打开的分类，不需要持久化为跨启动偏好。

## 图像生成

### 首屏

首屏只展示：

- 当前使用的服务商和模型。
- 已保存的服务列表。
- 每个服务的简短配置结论，例如“已配置”、“缺少 API Key”或“尚未配置”。
- “添加服务”操作。

不在首屏同时展开多套 API 表单。

### 服务详情

点击服务列表项或“添加服务”后，右侧内容区进入服务详情，同时保留左侧一级导航。详情页提供“返回图像生成”，并根据服务类型显示：

- API 地址。
- API Key。
- 模型。
- 该服务已有的其他必要字段。

多服务能力继续保留，但不将一个服务的配置失败包装成整个图像生成系统不可用。

## Codex 集成

### 产品定位

该页是一次性接入和使用引导，不是持续连接面板。因此页面不提供“允许 Codex 连接”开关，不使用“已连接”作为总体状态。

### 安装或更新

页面提供一条可复制的终端指令。指令内容根据检测结果变化：

- 缺少依赖时提供安装指令。
- 版本过旧时提供更新指令。
- 安装损坏或权限异常时提供修复指令。
- 全部齐全时显示“环境已准备好”，重新安装放入次级操作。

页面本身不自动执行指令，也不要求 CoreStudio 获得额外的系统修改权限。

### 环境检测

页面打开时自动检测，并保留“重新检测”操作。三项结果直接列出，不收入折叠区：

1. CoreStudio CLI 是否可用。
2. Codex 可发现的 CoreStudio Skill 或插件是否存在。
3. 安装版本、兼容性和本机会话发现能力是否正常。

失败项直接显示缺失原因，并与上方的安装、更新或修复指令保持一致。

### 在 Codex 中开始

页面提供一段可复制的自然语言指令，默认文案为：

> 打开当前 CoreStudio 项目

用户将这段指令粘贴到 Codex 对话即可。设置页不直接拉起 Codex，不创建对话，也不自动附加任务。

## 实验性功能

### 首屏

首屏提供统一说明：实验性功能需要手动开启，行为和配置可能继续调整。

第一版只展示一项实验性功能：**外部 Agent（ACP）**。

ACP 使用唯一开关：

- 关闭时，只显示功能名称、一句用途说明和开关。
- 开启时，在同页显示“Agent 类型”选择和“高级配置”入口。
- 删除现有 ACP 详情内部的第二个启用开关。
- 不在首屏长篇解释直接输入、ACP、CLI 和 Local Bridge 的关系。

ACP 开启后，CoreStudio 底部输入区自然出现“ACP Agent”模式；关闭后隐藏该模式。

### 高级配置

点击“高级配置”后，右侧内容区进入 ACP 详情，显示：

- 命令。
- 参数。
- 工作目录。
- 任务说明模板。
- 保存操作。
- 调试记录入口。

调试记录继续属于高级内容，不返回实验性功能首屏。

## 保存与离开规则

- 开关切换立即生效。
- 当前图像生成服务的选择立即生效。
- API Key、模型、API 地址、ACP 命令、参数、工作目录和任务模板等表单需要明确点击“保存”。
- 离开存在未保存修改的详情页时，显示“放弃修改”和“继续编辑”。
- 设置窗口不提供全局保存按钮。

## 画布上的 Codex 入口

移除画布右下角现有的 Codex 协作状态按钮及浮层。

Codex 集成不再作为需要长期监控的“连接状态”出现在画布上。安装、检测、修复和使用指令都回到应用设置。不新增无状态的 Codex 快捷按钮作为替代。

## 异常、加载与空状态

### 图像生成

- 没有已配置服务时，首屏只提示“尚未配置图像生成服务”并提供“添加服务”。
- 当前服务缺失 Key 或模型时，在对应服务行显示缺失项，不用全页错误。
- 保存失败时保留用户已输入内容，并在表单内显示可重试的错误。

### Codex 集成

- 检测进行中使用三行稳定骨架或“正在检测”文字，不让页面高度跳变。
- 单项检测失败不阻塞其他结果。
- 检测本身异常时显示“无法完成检测”和“重新检测”，不伪造“已准备好”。
- 没有当前项目时，使用指令仍可复制；Codex 在执行时负责让用户选择最近项目。

### 实验性功能

- ACP 不可编辑时，保留当前状态并说明原因，不清空已有配置。
- ACP 保存失败时保留草稿，并将错误放在保存操作附近。

## 可访问性与响应式

- 设置窗口必须支持 Escape 关闭；有未保存修改时，Escape 先进入离开确认。
- 左侧分类导航支持键盘聚焦和清楚的当前项状态。
- 复制指令、重新检测、开关和服务列表项都需要可读文字或 `aria-label`。
- 检测结果不只依赖颜色，同时使用文字或图标表达。
- 宽度不足时，左侧导航可收窄，但不将三个分类内容重新堆叠成一个长页。
- 文字、输入框和按钮继续使用 CoreStudio / Excalidraw 既有字号与控件尺寸，不通过局部继承放大或缩小。

## 验收标准

1. 用户打开“应用设置”后，可以稳定地在三个一级分类间切换，不再滚动一个混合长页。
2. 图像生成首屏能看到当前服务和所有已保存服务，但不同时显示多套表单。
3. Codex 集成页不包含权限开关、“已连接”状态、Board 链接或 CLI 环境变量快捷操作。
4. Codex 集成页能复制与当前检测结果一致的安装、更新或修复指令。
5. Codex 集成页直接列出三项真实检测结果，并可复制“打开当前 CoreStudio 项目”。
6. ACP 在实验性功能中只有一个开关，默认不显示命令、参数、工作目录、任务模板和调试记录。
7. 表单修改不会在未点击保存时被写入，离开未保存表单时会进行确认。
8. 画布右下角不再出现 Codex 协作状态按钮和浮层。
