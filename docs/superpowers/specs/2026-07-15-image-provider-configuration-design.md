# 图像生成服务配置重构设计

日期：2026-07-15
状态：已完成方案讨论，待书面复核

## 1. 背景

CoreStudio 当前同时在画布输入区和应用设置中暴露服务商、API Key 与模型配置，形成了两个配置入口。设置页还长期展示未配置的服务商，画布则保留了“连接与自定义模型”等已经失效的提示，使用户难以判断应该在哪里完成配置。

本次重构只解决图像生成服务的配置与使用边界，不调整 Codex 集成或 Excalidraw 基线。

## 2. 目标

建立唯一且清楚的职责边界：

- 「应用设置 → 图像生成」负责决定有哪些服务和模型可以使用。
- 画布输入区只读取已经保存的配置，负责选择服务、模型和本次生成参数。
- 未完成配置的服务不进入服务列表，也不出现在画布输入区。
- 现有有效配置自动迁移，用户不需要重新填写。
- 第一版支持经过验证的内置服务商，以及一个受限的 OpenAI 兼容服务。

## 3. 不在本次范围内

- 不建立任意插件式服务商系统。
- 不承诺维护一份“完整且永久不变”的图像模型服务商名单。
- 不在画布输入区编辑 API Key、Base URL 或模型目录。
- 不允许创建多个 OpenAI 兼容服务实例。
- 不调整第三方服务商侧的 API Key 或账户状态。
- 不在本轮完成安装包发布。

## 4. 核心模型

### 4.1 服务商目录

CoreStudio 内置服务商目录，用于描述每个受支持服务商的固定能力：

- 服务商标识和显示名称
- 接口适配器
- 内置模型目录
- 配置字段
- 模型支持的生成能力

第一版沿用并整理当前已经接入的服务商：Gemini、ZenMux、fal.ai、即梦 / Seedream、OpenAI、OpenRouter；另增加一个「OpenAI 兼容服务」定义。服务商目录只代表 CoreStudio 能配置的类型，不代表用户已经配置了这些服务。

服务商生态和模型名单会持续变化，因此采用“经过验证的内置目录 + 聚合服务模型目录 + 已知服务商下的自定义模型 + 一个受限兼容服务”的组合，不用静态名单假装覆盖全部生态。

### 4.2 已配置服务

只有满足必填条件并成功保存的服务才构成“已配置服务”。已配置服务是设置首页和画布输入区共同读取的唯一数据源。

一项配置至少包含：

- 服务商标识
- API Key
- 默认模型
- 可选的自定义模型
- 配置状态和必要的兼容版本信息

第一版中，每个内置服务商只能保存一项配置；OpenAI 兼容服务也只能保存一项配置。这样可以继续使用稳定的服务商标识，避免在本轮引入动态实例 ID 和大规模项目数据迁移。

### 4.3 默认服务与上次使用记录

设置保存全局默认服务和各服务的默认模型。画布可以单独记住用户上次使用的服务与模型，但这只是使用偏好，不会反向修改设置。

恢复优先级为：

1. 上次使用的服务和模型仍然存在时，恢复上次选择。
2. 否则使用设置中的默认服务和默认模型。
3. 默认服务失效时，使用已配置列表中的第一项。
4. 没有已配置服务时，进入未配置状态。

## 5. 设置页交互

### 5.1 图像生成首页

设置首页只展示已配置服务，不再把所有内置服务商及“缺少 API Key”状态长期铺开。

每项只显示：

- 服务名称
- 默认模型
- 已配置状态

点击服务项进入编辑页。页面提供唯一的「添加服务」按钮。没有已配置服务时，显示简短空状态和「添加服务」。

### 5.2 添加服务

添加采用设置内容区内的逐级页面，不叠加第二层弹窗：

1. 点击「添加服务」。
2. 选择一个尚未配置的内置服务商，或选择「OpenAI 兼容服务」。
3. 填写该服务所需参数。
4. 校验通过后保存并返回服务列表。

已经配置且不允许重复的服务商不再出现在可添加列表中。

### 5.3 内置服务配置

内置服务商的基础字段为：

- API Key，必填
- 默认模型，必填
- 自定义模型，可选

ZenMux 作为第一版的基准实现。其 Base URL 和已验证的接口适配器由 CoreStudio 内置，不要求用户填写。

自定义模型包含：

- 模型 ID，必填
- 显示名称，可选；为空时使用模型 ID
- 接口类型，只能从该服务商已经支持的适配器中选择
- 能力声明，包括是否支持参考图、尺寸或比例等

能力声明控制画布显示哪些本次生成参数，不允许用户填写任意请求模板或脚本。

### 5.4 OpenAI 兼容服务

第一版只允许配置一个 OpenAI 兼容服务，字段为：

- 服务名称，必填
- Base URL，必填
- API Key，必填
- 模型 ID，必填
- 模型能力，包括参考图、尺寸和比例支持情况

该服务只支持 CoreStudio 明确定义的 OpenAI 兼容图像请求，不提供任意协议、请求体模板或插件扩展。

### 5.5 保存、编辑和删除

- 必填项不完整时禁止保存，不创建半配置服务。
- 保存第一个服务后，自动将其设为默认服务。
- 后续新增服务不自动替换默认服务。
- 编辑 API Key、默认模型或自定义模型后，画布重新读取配置。
- 删除服务需要普通确认，文案只说明该服务将不再出现在画布服务列表中。
- 删除非默认服务不改变当前默认服务。
- 删除默认服务后，有其他服务时将列表第一项设为新默认服务。
- 删除最后一个服务后进入未配置状态。

## 6. 画布输入区

画布输入区是配置的消费端，不是配置入口。

- 服务商下拉框只显示已配置服务。
- 模型下拉框只显示当前服务中已经配置好的内置模型和自定义模型。
- 输入区可以修改服务、模型以及本次生成的尺寸、比例、参考图等运行参数。
- 输入区不提供添加服务、API Key、Base URL、自定义模型目录或接口类型编辑。
- 设置变更后，输入区重新读取同一份公开配置，不维护第二份服务和模型数据。

没有已配置服务时，输入区显示“尚未配置图像生成服务”和一个「打开应用设置」入口。该入口只是导航，不在画布中展开配置。用户已经输入的提示词、参考图和本次参数予以保留，配置完成返回后可以继续生成。

## 7. 配置刷新与数据流

数据流固定为：

1. Electron 设置存储保存密钥和完整服务配置。
2. 设置模块校验并生成不包含敏感密钥的公开配置。
3. 应用设置和画布输入区订阅同一份公开配置。
4. 画布提交生成任务时只传服务标识、模型和本次参数。
5. Electron 生成运行时根据服务标识读取对应密钥与适配器并发起请求。

公开配置只暴露服务是否可用、默认模型、可选模型和能力，不把 API Key 传入渲染层现有业务状态。沿用当前密钥保存与进程边界，不在本轮扩大敏感数据暴露范围。

## 8. 迁移规则

启动时将旧设置转换为新的已配置服务：

- 存在有效 API Key 且默认模型可解析的内置服务，迁移为已配置服务。
- 同步保留其自定义模型和最近一次验证状态中仍适用的信息。
- 没有 API Key 的旧服务记录不进入已配置列表。
- 旧的当前服务仍已配置时，作为默认服务；否则选择第一项已配置服务。
- 旧的画布上次选择仍有效时予以保留，否则按新的恢复优先级回退。

迁移应可重复执行，不能重复创建服务或破坏已经转换的数据。

## 9. 错误处理与文案

- 删除“连接与自定义模型”等已经失效的入口和提示。
- 没有已配置服务时，只提示前往应用设置。
- API Key 无效、接口无法访问、模型不存在和请求参数不受支持应分别显示。
- 校验失败时保留用户已经填写的配置内容。
- 自定义模型生成失败时不自动删除配置，提示用户在设置中检查模型 ID、接口类型和能力声明。
- 运行中发现服务已经被删除或失效时，停止本次请求并刷新公开配置，不生成隐式的临时服务。

## 10. 测试与验收

### 10.1 设置页

- 空状态只能通过「添加服务」开始配置。
- 已配置列表不显示缺少 API Key 的服务。
- 添加、编辑和删除内置服务符合默认服务规则。
- 同一内置服务和 OpenAI 兼容服务不能重复添加。
- 必填字段、Base URL 和模型 ID 校验正确。
- ZenMux 的内置模型与自定义模型可以保存并重新读取。

### 10.2 画布输入区

- 只显示已配置服务和模型。
- 设置新增、编辑或删除服务后能够刷新。
- 切换模型后只显示该模型支持的本次生成参数。
- 输入区不存在 API Key、Base URL 或模型目录编辑入口。
- 无服务时提示正确，且提示词、参考图和本次参数不会丢失。

### 10.3 运行时和迁移

- ZenMux 不同适配器的请求构造保持正确。
- OpenAI 兼容服务按受限协议构造请求。
- 旧有效配置自动迁移，旧无效配置不会污染已配置列表。
- 删除默认服务、删除最后一个服务和配置运行中失效时回退正确。
- API Key 不进入公开配置或画布持久化状态。

## 11. 完成标准

- 设置页只有一个明确的服务配置入口。
- 画布输入区只消费已配置服务，不再形成第二个配置入口。
- ZenMux、内置服务自定义模型和单个 OpenAI 兼容服务形成完整闭环。
- 旧配置迁移和关键交互具有自动化测试。
- 开发版完成界面和生成链路验证后，再单独决定打包时机。
